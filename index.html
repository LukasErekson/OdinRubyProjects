Ruby Projects

Ruby exercise solutions and projects from The Odin Project

This is a compilation of all the exercises and projects I've done as part of The Odin Project's Ruby Programming course. The repository contains all my solutions to the regular ruby exercises, but I'm particularly proud of my work on the projects.

This page is a reflection on the important concepts I learned during the projects rather than the details of the projects themselves. The source code can be found on my GitHub.

I tried to go above and beyond for each of the projects, so I hope you can take the time to see them in action. I tried to learn and implement a new best-practice in each of my successive projects. For instance, several projects come with their own unit tests using RSpec while others also have their own RDoc documentation.

User Input & Serialization

Creating games to play in the terminal

Several of the early projects in The Odin Project are simple games meant to be played in the terminal. This lent itself well to learning and applying the syntax for parsing user input and saving files. Two games that I feel I implemented really well are Tic-Tac-Toe and Mastermind. The source code for each of these games as well as a playable version online are available as replit.io repositories.

Of the two, Mastermind is my favorite. I was able to use a gem that allowed for colored output, and I feel like I handled errors and exceptions well. I also implemented a basic algorithm that allows the player to input a code and the computer to "guess" their code. Feel free to play a couple of rounds of the game by clicking here. Tic-Tac-Toe, however, is a cleaner, more standard project as I went back and refactored the code when I was learning more about automated testing.

Testing, Testing, 1 2 3...

Learning RSpec to verify solutions and drive development

Using the spec files provided for the exercises and looking up the documentation, I started to create RSpec test files for most of the projects. I wanted to practice writing unit tests and try RSpec's "behavior driven development" methodology. My previous experience with Python during my undergraduate research (see my Rootfinding fork) instilled in me the importance of creating and using unit tests. I created my own unit tests for nearly all of the projects before the course formally covered testing.

Out of them all, I'm most proud of the work I did with Linked List. It was the first project that I really tried to do Test-Driven Development. I wrote out the spec file that tested for the specific behaviors and outputs for my LinkedList class before implementing any of the methods. Doing so gave me not only a clear end-goal, but it also helped me identify errors when handling the edge cases that I may not have caught otherwise. I had a similar experience when implementing the Binary Search Tree project.

Additionally, I started to learn a little bit about generating documentation using RDoc. You can find the source code for the classes and the spec files by clicking here. If you're interested in browsing the generated documentation, that can be found here.

Data Structures & Algorithms

Implementing algorithms - from pseudocode to Ruby

The Odin Project provided me with several opportunities to implement algorithms from pseudocode. For instance, my Mastermind implementation includes an algorithm that allows the program to systematically determine the user's secret code. The sections on recursion, including the Binary Search Tree, also gave me a lot of opportunity to consider carefully how to translate an abstract structure and algorithm into clean, usable code.

As I have a lot of experience implementing data structures and alogrithms from my undergraduate coursework in Python and C++, I felt very capable of breaking-down, implementing, and analyzing algorithms in Ruby.
